1. Ringkasan project & ide besar

Program ini menghitung besaran geometri untuk tiga jenis “bentuk”:

  • Segi empat sembarang (2D) — hanya di bidang datar: luas, keliling. Volume = 0; luas permukaan = luas bidang.
  • Limas — alas berbentuk segi empat sembarang + tinggi limas + tinggi sisi tegak tiap rusuk alas (untuk luas sisi miring).
  • Prisma tegak — alas segi empat sembarang + tinggi prisma (sisi tegak tegak lurus alas).

Multithreading: untuk demo, setiap bentuk diproses di thread sendiri (GeoProcessor implements Runnable). Ada jeda (Thread.sleep) supaya terlihat bahwa thread bisa “bergantian” jalan. Log dicetak dengan synchronized agar baris tidak saling menyisip huruf.

Ide OOP utama: tiga bentuk itu berbeda rumusnya, tapi bagi pemanggil (main, prosesor thread, GUI) semuanya hanya “sesuatu yang bisa hitungLuas() …”. Itu dicapai dengan interface Geometri dan polimorfisme (variabel bertipe Geometri bisa menunjuk ke objek konkret mana pun yang implements Geometri).

Untuk laporan: tulis satu paragraf tujuan (apa yang dihitung), satu paragraf desain OOP (interface + inheritance + overriding), satu paragraf concurrency (thread per bentuk, join, sinkronisasi log), dan satu paragraf antarmuka (konsol vs Swing).

2. Konsep OOP dasar (bahasa sederhana)

Sebelum “lima pilar”, ini istilah yang muncul terus di project:

Class
Cetak biru / definisi: nama field (data) dan method (perilaku). Contoh: class SegiEmpatSembarang mendefinisikan sisi a,b,c,d dan cara menghitung luas.
Object (objek)
Isi nyata di memori saat program jalan: new SegiEmpatSembarang(3,4,3,4, …) menghasilkan satu segi empat dengan angka tertentu.
Interface
Kontrak: daftar method yang harus ada, tanpa isi rumus (di Java: method di interface, implementasi di class). Geometri memaksa semua bentuk punya hitungLuas(), hitungKeliling(), dst.
implements Geometri
Class “menandatangani kontrak”: ia berjanji mengisi semua method Geometri. Tiga class bentuk melakukan ini dengan rumus berbeda.
Inheritance (“is-a” / pewarisan)
LimasSegiEmpatSembarang dan PrismaSegiEmpatSembarang extends SegiEmpatSembarang: mewarisi sisi, sudut, getter, serta bisa memanggil super.hitungLuas() / super.hitungKeliling(). Bangun 3D dimodelkan sebagai “segi empat sembarang + dimensi tambahan” (tinggi limas/prisma, tinggi sisi miring untuk limas).
Package
Semua file punya package geometri; supaya class saling kenal tanpa import panjang dalam satu modul kecil.

Hirarki bentuk: SegiEmpatSembarang (induk, 2D) → LimasSegiEmpatSembarang / PrismaSegiEmpatSembarang (anak, 3D). Semua implements Geometri. GUI memakai pewarisan lain: GeometriFrame extends JFrame.

3. Penjelasan tiap class & file

Berikut cocokkan dengan file di src/geometri/ saat membaca kode atau menulis laporan.

Geometri.java — interface

Hanya deklarasi method: hitungLuas, hitungKeliling, hitungVolume, hitungLuasPermukaan, getInfo. Tidak ada field. Peran: menyatukan cara memanggil bentuk 2D dan 3D dalam satu tipe referensi.

SegiEmpatSembarang.java — bentuk 2D

  • Data: private final double a,b,c,d (sisi berurutan) dan sudutA…sudutD (derajat). final artinya setelah konstruktor selesai, nilai tidak diganti — cocok untuk model matematika yang stabil.
  • hitungKeliling: jumlah keempat sisi.
  • hitungLuas: cari panjang diagonal antara ujung sisi a dan b memakai cosinus pada sudut B (lihat bagian matematika), lalu bagi segi empat jadi dua segitiga yang sama-sama berbagi diagonal itu; tiap segitiga luasnya Heron.
  • hitungVolume: selalu 0.0 (bangun planar).
  • hitungLuasPermukaan: sama dengan luas (untuk 2D).
  • Getter getSisiA … getSisiD: diwariskan ke subclass limas/prisma untuk menghitung luas sisi miring (panjang rusuk alas per sisi).

LimasSegiEmpatSembarang.java — limas alas segi empat

  • Deklarasi: extends SegiEmpatSembarang implements Geometri — mewarisi sisi/sudut dari induk; field tambahan: tinggiLimas dan empat tinggiSisi* (tinggi segitiga sisi miring di atas tiap rusuk).
  • Konstruktor: super(a,b,c,d,sudutA,…) lalu menyimpan tinggi; dipanggil langsung dari ShapeGenerator.
  • hitungLuas / keliling: @Override — delegasi ke super.hitungLuas() / super.hitungKeliling() (luas/keliling alas).
  • hitungVolume: @Override — ⅓ × super.hitungLuas() × tinggiLimas.
  • hitungLuasPermukaan: luas alas (super) + empat segitiga: 0.5 * (getSisiX() * tinggiSisiX).

PrismaSegiEmpatSembarang.java — prisma tegak

  • Deklarasi: extends SegiEmpatSembarang implements Geometri; field tambahan: tinggiPrisma.
  • Konstruktor: super(...) + tinggiPrisma.
  • hitungVolume: super.hitungLuas() * tinggiPrisma.
  • hitungLuasPermukaan: 2 * super.hitungLuas() + super.hitungKeliling() * tinggiPrisma.

ShapeGenerator.java — data uji acak

  • Class final dengan konstruktor private: tidak dibuat instansnya; hanya method statis generateRandomShapes(min, max).
  • Mengembalikan List<Geometri> berisi campuran ketiga jenis bentuk (random).
  • Membantu demo multithreading tanpa mengetik bentuk manual.

GeoProcessor.java — satu thread = satu bentuk

  • implements Runnable: punya method run() yang dipanggil saat Thread.start().
  • Menyimpan referensi Geometri shape — inilah polimorfisme: tidak peduli bentuknya apa, cukup panggil shape.hitungLuas() dll.
  • Overloading konstruktor: versi tiga argumen (log ke konsol) vs empat argumen dengan Consumer<String> logSink untuk GUI.
  • logLine: synchronized (LOCK) memastikan hanya satu thread pada satu waktu yang menulis satu baris penuh ke System.out atau ke consumer.
  • Exception: InterruptedException saat sleep — menangkap, mencetak error, lalu Thread.currentThread().interrupt() (praktik baik: menjaga flag interrupt).

Main.java — mode konsol

  • Memanggil ShapeGenerator.generateRandomShapes dengan konstanta min/max.
  • Membuat array Thread, tiap thread membungkus GeoProcessor untuk satu elemen list.
  • start() lalu join(): program utama menunggu semua selesai sebelum printSummary.
  • printSummary memakai synchronized (System.out) agar ringkasan tidak bertabrakan dengan sisa output.

GeometriFrame.java — mode GUI (Swing)

  • Parameter: spinner min/max jumlah bentuk, min/max delay (ms).
  • Pisah thread: klik “Jalankan” membuat thread koordinator (runBatch) — supaya EDT (thread UI) tidak diblokir saat join.
  • Worker thread memakai GeoProcessor(..., toEdt) dengan toEdt = lambda yang meneruskan setiap baris log ke SwingUtilities.invokeLater(() -> appendLogLine(line)) — aturan Swing: mengubah komponen UI hanya dari EDT.
  • Setelah semua join, mengisi JTable lewat invokeLater lagi. Kolom volume: — hanya untuk instance 2D murni (getClass() == SegiEmpatSembarang.class), karena limas/prisma juga extends segi empat.
  • Tombol “Hentikan” memanggil interrupt() pada worker threads yang masih hidup.

4. Alur program — langkah demi langkah

4.1 Mode konsol (Main.main)

  1. Cetak judul.
  2. Panggil ShapeGenerator.generateRandomShapes(JUMLAH_BENTUK_MIN, JUMLAH_BENTUK_MAX) → dapat List<Geometri> berisi N objek (N acak di rentang tersebut).
  3. Buat array Thread[] sepanjang N. Untuk indeks i: buat GeoProcessor(shapes.get(i), "Thread-"+(i+1), delayMs) dengan delay acak, lalu threads[i] = new Thread(processor, "GeoWorker-"+(i+1)).
  4. Loop: t.start() untuk semua thread — urutan selesai tidak dijamin.
  5. Loop: t.join() — thread utama berhenti di sini sampai tiap worker selesai (atau interrupt).
  6. printSummary(shapes): di thread utama, loop lagi tiap Geometri g dan panggil method hitung (bukan lewat GeoProcessor lagi) untuk menampilkan ringkasan angka.

Jadi ada dua fase: (A) proses paralel dengan log panjang per thread; (B) ringkasan singkat sequential di akhir.

4.2 Mode GUI (GeometriFrame)

  1. User mengisi spinner → klik Jalankan. Validasi min ≤ max, delay min ≤ max.
  2. UI dinonaktifkan sementara (tombol jalankan off, progress “Berjalan…”).
  3. Thread koordinator runBatch jalan: generate list bentuk sama seperti konsol.
  4. Untuk tiap bentuk: buat GeoProcessor dengan Consumer yang meneruskan log ke EDT.
  5. start() semua worker → join() semua (masih di thread koordinator, bukan EDT).
  6. Setelah join: invokeLater untuk mengisi tabel ringkasan.
  7. finally: kembalikan tombol dan progress ke “Siap”.
Inti untuk laporan: jelaskan perbedaan worker thread (hitung + log) vs thread koordinator (GUI) vs EDT (menggambar ulang JTextArea dan JTable). Gambarkan: worker tidak boleh menyentuh komponen Swing langsung.

5. Lima pilar OOP — penjelasan mendalam + bukti di kode

Urutan di bawah mengikuti modul umum “lima pilar”. Tiap pilar: apa itu → kenapa dipakai → bukti konkret di project.

5.1 Encapsulation & information hiding

Pengertian sederhana: detail internal objek disembunyikan; dunia luar berinteraksi lewat antarmuka terbatas (method publik, getter), bukan dengan mengubah data sembarangan.

Di project: sisi dan sudut segi empat private final di induk — subclass limas/prisma mengakses panjang sisi hanya lewat getter getSisiA() … yang diwariskan. Di GeoProcessor, method displayProcessingStatus / displayResult adalah private.

Kalimat untuk dosen: “Kami membungkus data bentuk di dalam class dan hanya mengekspos apa yang dibutuhkan pemanggil, sesuai interface Geometri.”

5.2 Inheritance (pewarisan)

Pengertian: class anak mewarisi kode/data class induk; relasi “is-a”.

Di project: LimasSegiEmpatSembarang dan PrismaSegiEmpatSembarang extends SegiEmpatSembarang — mewarisi field sisi/sudut, getter, serta memakai super untuk luas/keliling alas. GeometriFrame extends JFrame (pewarisan GUI). Polimorfisme tetap lewat Geometri; subclass juga bisa dirujuk sebagai SegiEmpatSembarang jika perlu data alas.

Kalimat untuk dosen: “Limas dan prisma adalah segi empat sembarang yang diperluas dengan tinggi — kami pakai inheritance supaya rumus alas tidak diduplikasi; volume dan luas permukaan di-override di class anak.”

5.3 Overloading

Pengertian: nama method (atau konstruktor) sama, daftar parameter berbeda.

Di project: dua konstruktor GeoProcessor (dengan/tanpa Consumer<String> untuk log GUI).

5.4 Overriding & polymorphism

Overriding: class implementasi menulis ulang method dari kontrak interface dengan isi sendiri.

Polimorfisme: variabel bertipe Geometri bisa menunjuk ke SegiEmpatSembarang, Limas…, atau Prisma…. Saat memanggil shape.hitungVolume(), JVM memilih implementasi sesuai objek sebenarnya — pemanggil tidak perlu if (shape instanceof …) untuk rumus dasar.

Di project: loop di GeoProcessor.run dan loop ringkasan di Main / tabel GUI semuanya memakai referensi Geometri.

5.5 Multithreading

Pengertian: lebih dari satu jalur eksekusi; Runnable + Thread; start / sleep / join; sinkronisasi saat sumber daya bersama (di sini: saluran log).

Di project: GeoProcessor implements Runnable; synchronized (LOCK) pada logLine; join di konsol dan koordinator GUI; penanganan InterruptedException.

6. Matematika dari nol (yang dipakai program)

Bagian ini untuk memahami mengapa rumus di kode seperti itu. Tidak perlu lulus kalkulus; cukup geometri SMA dan sedikit trigonometri.

6.1 Keliling segi empat

Jika empat sisi berurutan a, b, c, d membentuk satu putaran, keliling = a + b + c + d. Itu langsung di hitungKeliling.

6.2 Luas segitiga (Heron)

Jika sebuah segitiga punya sisi panjang x, y, z (ketiganya diketahui), luasnya bisa dihitung tanpa sudut:

  • Setengah keliling: s = (x + y + z) / 2
  • Luas: sqrt(s * (s-x) * (s-y) * (s-z)) asalkan segitiga valid (di kode, jika dalam akar ≤ 0, dikembalikan 0).

Ini dipakai di method statis heron(x,y,z) pada SegiEmpatSembarang.

6.3 Aturan kosinus untuk diagonal

Bayangkan segi empat dengan sisi pertama a dan b bertemu di sebuah vertex, dan sudut di dalam di vertex itu adalah sudutB (sesuai penamaan di kode: sudut antara sisi a dan b).

Panjang diagonal d yang menutup segitiga dari sisi a dan b memenuhi:

d² = a² + b² − 2ab cos(θ) dengan θ dalam radian — di Java: Math.toRadians(sudutB) lalu Math.cos.

Setelah d diketahui, segi empat dipotong diagonal itu jadi dua segitiga: (a, b, d) dan (c, d, d) — perhatikan segitiga kedua memakai sisi c, d, dan diagonal yang sama d. Luas total = Heron segitiga 1 + Heron segitiga 2. Itu persis alur hitungLuas().

Kenapa ada sudut kalau kita sudah punya sisi? Empat sisi saja belum tentu menentukan satu bentuk unik (bisa “flex”). Program memakai sudutB untuk menentukan diagonal utama yang memecah luas; sudut lain ikut disimpan untuk konsistensi data dan teks getInfo.

6.4 Volume & luas permukaan limas

  • Volume: luas alas × tinggi × ⅓. Luas alas dari induk: super.hitungLuas().
  • Luas permukaan: luas alas + jumlah empat segitiga sisi tegak. Luas segitiga = ½ × alas segitiga × tinggi. “Alas segitiga” tiap sisi = rusuk alas (getSisiA() …); “tinggi” = tinggiSisiA … yang diinput/dibangkitkan.

6.5 Volume & luas permukaan prisma tegak

  • Volume: luas alas × tinggi prisma (tegak lurus alas).
  • Luas permukaan: dua tutup (atas+bawah) + selimut persegi panjang yang “dipelintir”: keliling alas × tinggi prisma.

6.6 Angka di tabel untuk bentuk 2D

Volume bangun 2D tidak terdefinisi; di kode nilainya 0, di GUI ditampilkan — agar pembaca tidak mengira volume 0 “valid”.

Narasi detail: menjelaskan dynamic dispatch tanpa istilah berat
  1. “Di GeoProcessor, variabelnya bertipe Geometri. Itu artinya: kita hanya janji punya method hitung — tidak janji bentuknya limas atau prisma.”
  2. “Saat program jalan, JVM melihat objek sebenarnya di heap: kalau isinya limas, yang dipanggil rumus limas; kalau prisma, rumus prisma.”
  3. “Limas dan prisma extends segi empat: luas/keliling alas lewat super.hitungLuas() — tidak menduplikasi rumus Heron di class anak.”
  4. “Untuk laporan: ini contoh overriding dari kontrak interface + polimorfisme lewat referensi induk.”

Loncat ke kode: bandingkan SegiEmpatSembarang.java, LimasSegiEmpatSembarang.java, PrismaSegiEmpatSembarang.java pada method dengan anotasi @Override.

Narasi detail: kenapa GUI “muter” thread lebih banyak
  1. “Swing punya EDT — satu jalur yang aman mengubah komponen. Kita tidak boleh blokir EDT dengan join panjang, makanya runBatch jalan di thread lain.”
  2. “Worker thread juga tidak boleh langsung sentuh JTextArea; lewat Consumer kita serahkan ke invokeLater supaya append teks terjadi di EDT.”
  3. “Setelah semua worker selesai, koordinator mem-schedule pengisian tabel ke EDT juga — karena mengubah model tabel harus konsisten dengan aturan UI.”

9. Cheat sheet Q&A (untuk presentasi / bimbingan)

  • Di mana “segi empat sembarang” pada limas/prisma? Diwariskan dari SegiEmpatSembarang (sisi/sudut di induk); luas alas = super.hitungLuas() atau getter getSisiA() … getSisiD().
  • Kenapa ada sudut dan tinggi sisi limas? Sudut+sisi membentuk alas; tinggi sisi tegak dipakai rumus luas sisi miring Σ ½ sᵢhᵢ.
  • Kenapa volume 2D “—” di GUI? Bangun planar tidak punya volume; tampilan membedakan dari angka 0.
  • Sinkronisasi untuk apa? Satu saluran output; tanpa lock, baris log bisa tercampur.
  • Polimorfisme satu kalimat? Satu tipe Geometri, banyak implementasi; pemanggilan method mengikuti objek nyata.
